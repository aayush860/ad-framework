Automated Load Testing Setup

Docker Compose orchestrates multiple containers, each running Locust scripts to generate metrics and simulate load automatically

▶️ How It Works

1

Docker Compose Orchestration

docker-compose.yml defines multiple services (Germany, Japan, Brazil containers). Each service spins up an isolated Docker container with its own OpenVPN configuration.

2

Automated Script Execution

Each container automatically runs simulator.py (or Locust script) to generate metrics. No manual intervention needed - containers start generating load immediately on deployment.

3

Geographic Load Distribution

Each container connects through country-specific OpenVPN tunnels, routing all traffic to appear from that geographic location. This simulates real-world distributed load patterns.

⚙️ Configurable Parameters

⚡ Load Intensity

INTERVAL: 1-60 seconds
USERS: 1-1000
SPAWN_RATE: 1-100/sec

Adjust request frequency and concurrent users in docker-compose.yml environment variables

🔐 OpenVPN Count

Range: 0 to 90 VPN configs
Per Container: 1 active VPN
Total Available: 90 countries

Select from 90 available country VPN configs. Scale from 0 containers (no load) to 90 containers (maximum geographic distribution)

📦 Container Scaling

Current: 3 containers
Maximum: 90 containers
Deploy: docker-compose up --scale

Add/remove container instances using Docker Compose scaling. Each new container = new geographic location with independent load generation

Quick Start Commands

# Start all containers with default config
docker-compose up -d

# Scale to 10 containers (10 different locations)
docker-compose up -d --scale germany=10

# Stop all load generation
docker-compose down
